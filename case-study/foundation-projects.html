üöÄ Space War

Android 2D Shooter Collision Optimization

Overview

A vertical endless shooter where the player battles waves of enemy ships with varying health, damage, and attack patterns.

Core Systems Implemented

  • Enemy spawning and behavior control
  • Weapon, damage, and health systems
  • Multi-entity collision handling
  • Player and enemy destruction logic

Key Challenge

Initially, collision handling was implemented using multiple OnCollisionEnter checks across scripts for: bullets, players, enemies, and bullet-vs-bullet interactions. This approach quickly became difficult to scale and inefficient.

Solution & Learning

I redesigned the system using Unity‚Äôs Layer Collision Matrix, filtering interactions at the engine level instead of script-level checks.

  • Cleaner and more maintainable code
  • Reduced unnecessary collision checks
  • Improved performance and scalability

üèÉ Endless Runner

Android Optimization DOTween

Overview

A simple endless runner inspired by Chrome‚Äôs Dino Run, focused on smooth visuals and efficient runtime performance.

Core Systems Implemented

  • Endless ground and background movement
  • Player jumping and obstacle interaction
  • DOTween-based smooth transitions

Key Challenge

Traditional endless runner designs rely on frequent spawning and destruction of environment objects, which can cause CPU and GC overhead on mobile devices.

Solution & Learning

Instead of spawning objects, I implemented texture offset scrolling for ground and background materials.

  • No runtime object instantiation
  • Smoother visual performance
  • Lower CPU and memory usage
