My Journey

Hi, I’m Nikhil Chaudhary, a Multiplayer Unity Game Developer with 2+ years of experience building games for Android, WebGL, and Windows.

My journey started with curiosity about how games work behind the scenes — not just visuals, but systems, logic, networking, and performance. Over time, this curiosity evolved into a deep passion for clean architecture, scalable multiplayer systems, and polished gameplay experiences.

How I Started

I began my development journey by learning C# and Unity, building small single-player prototypes and gameplay mechanics. Very soon, I realized that writing code that simply “works” is not enough — it needs to be maintainable, testable, and scalable.

Transition into Multiplayer & Backend

To challenge myself further, I moved into multiplayer game development using Photon PUN, real-time synchronization, room-based architecture, and player state management.

Alongside this, I integrated backend systems using Firebase, Node.js, Express.js, and MongoDB for player data, progression, and cloud storage.

What I Focus On Today

  • ✔ Multiplayer gameplay architecture (Photon PUN / room-based systems)
  • ✔ Clean architecture patterns (MVP, State Machine, Strategy, Singleton)
  • ✔ Modular AI systems & scalable enemy behaviors
  • ✔ Backend-integrated game data systems
  • ✔ Performance optimization & network data optimization
  • ✔ Polished UI/UX for game menus and lobbies

My Development Timeline

Phase 1

Learned Unity & C# fundamentals, gameplay mechanics, and basic UI systems.

Phase 2

Built single-player games and AI-driven systems using State & Strategy patterns.

Phase 3

My Goal

My goal is to become a high-impact game engineer who not only builds games, but also designs systems that can scale, be maintained by teams, and be taught to junior developers.

I strongly believe in learning deeply, building cleanly, and sharing knowledge.
